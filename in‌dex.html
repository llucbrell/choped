Chóped es un editor de código HTML, CSS y Javascript, con licencia MIT que hace uso de las librerías javascript; Codemirror y Zepto. Codemirror es el verdadero motor de este editor, un compendio de código realmente potente y que está especializado en edición de código. Visita su web si quieres saber más.

Apuntes

  • Se puede usar vía web, mediante una de estas dos urls Chóped //Chóped_for_lotsOf_Search o también en modo local descargando los archivos mediante el siguiente link de descarga

    Te recominedo el modo local porque siempre podrás trabajar con él y no dependerás de mi flojo servidor.

  • Recuerda, está en fase beta y eso implica que aún lo estoy probando. Así que, segurísimo tendrá algún bug o problema. Aunque sirve perféctamente para hacer webApps, puedes trabajar con él, es completamente funcional. Un ejemplo es que este about, está realizado exclusivamente con Chóped.
  • Si te interesa usar otros motores de búsqueda para la web (por defecto Chóped está configurado con Bing) encontrarás más incluidos en el código. He colocado tanto Google como DuckduckGo. Sólo tienes que cambiar el nombre contenido en la variable "buscador".

    Google funciona correctamente en modo local, aunque creo que no lo hace desde un servidor https. ¡Son así de majos!

    DuckduckGo no lo he testeado aún, aunque en un futuro espero convertirlo en el motor por defecto.

  • Chóped es un editor de código que funciona exclusivamente en el lado del cliente. Es decir, funcionará aunque no estés conectado a internet, no romperá nada de tu equipo, ni de tu servidor, en caso de que lo subas para usarlo.
  • He intentado que fuera potente y rápido, aunque todo esto seguro que es muy mejorable. Si resulta que tienes un computador con pocos recursos, o una baja conexión a internet, cambia la librería Zepto por jQuery. Debería bastar con eso para un funcionamiento decente.
  • Si decides usar diferentes archivos para tu código, CSSs o JSs externos al archivo principal, recuerda usar "file:///C:/PATH_COMPLETO" para referenciarlos, así no tendrás problemas con el preview.
  • Si tienes varias lonchas (cajetín) de código abiertas, recuerda que al apretar el botón guardar, sólo se guarda la loncha con el lateral naranja, y que se guarda en tu computadora, en futuras versiones se incluirá el guardado en localstorage, para no perder mortadela.
  • Si pones nombre al proyecto, no olvides incluir la extensión del archivo.. miproyecto.html, mi_proyecto.css, etc,.
  • Si tienes algún problema, sugeréncia o aportación al proyecto, escríbe a llucbrell@gmail.com
